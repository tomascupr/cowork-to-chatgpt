OpenAI's native desktop import is the right first step. It imports supported setup, projects, and
recent chats. This tool supplements that flow by converting the full locally persisted Cowork
archive into isolated Markdown documentation, including history older than the native 30-day chat
window.

```text
Claude Cowork                       Original workspace folder
──────────────────────              ─────────────────────────
Selected workspace folder  ─┐       existing files (preserved)
  CLAUDE.md                  ├────▶  AGENTS.md                 official instructions + memory
  memory/**/*.md             │       COWORK_INSTRUCTIONS.md    optional size overflow
Hidden session transcripts  ─┘       COWORK_HISTORY_*.md       supplemental Markdown
```

## What is native and what is not

The distinction is deliberate:

| File | OpenAI behavior |
|---|---|
| `AGENTS.md` | Documented Codex/ChatGPT desktop instruction file; imported Claude guidance is placed here |
| `COWORK_INSTRUCTIONS.md` | Ordinary Markdown overflow used only when `AGENTS.md` would exceed its documented limit |
| `COWORK_HISTORY_INDEX.md` | Ordinary Markdown index generated by this tool |
| `COWORK_HISTORY.md` | Ordinary Markdown history generated by this tool |

The `COWORK_` filenames have no special ChatGPT semantics. They are explicit names chosen to
avoid pretending otherwise and to avoid collisions with user-authored `MEMORY.md` or `HISTORY.md`.

For browser-based ChatGPT Projects, uploaded files are project sources and project instructions
are configured separately in Project settings.

## Quick start

First use the native ChatGPT desktop import under **Settings → Import**. Then install the full
local Cowork archive supplement:

```bash
uv tool install git+https://github.com/tomascupr/cowork-to-chatgpt.git

# Preview the workspace boundaries
cowork2chatgpt scan

# Add the supplemental context to original single-folder workspaces
cowork2chatgpt install
```

To run from a clone:

```bash
git clone https://github.com/tomascupr/cowork-to-chatgpt.git
cd cowork-to-chatgpt
uv sync --locked
uv run cowork2chatgpt scan
uv run cowork2chatgpt install
```

## What `install` changes

```text
your-existing-workspace/
├── your existing files...
├── AGENTS.md
├── COWORK_HISTORY_INDEX.md
└── COWORK_HISTORY.md
```

`install` is idempotent:

- Existing `AGENTS.md` content is preserved; a managed instruction section is added or refreshed.
- User-authored `MEMORY.md` and `HISTORY.md` files are not overwritten.
- Exporter-owned files from versions before 0.5 are safely migrated to the explicit `COWORK_`
  names.
- Unrelated `COWORK_` files are never overwritten.
- All other workspace files remain untouched.

Large histories become `COWORK_HISTORY_001.md`, `COWORK_HISTORY_002.md`, and so on.

## Workspace isolation

| Cowork session | Behavior |
|---|---|
| Same canonical selected-folder set | Grouped into one workspace |
| One selected folder | Eligible for in-place installation |
| Several selected folders | Kept as a distinct portable composite workspace |
| No selected folder | Kept as its own portable one-session workspace |
| Hidden global Cowork memory | Quarantined for selective review |

Composite sessions are never copied into each constituent folder. That would silently mix
contexts and make it impossible to tell which workspace owned the history.

## Portable export

Use `export` for browser ChatGPT Projects, backup, composite workspaces, or unassigned sessions:

```bash
cowork2chatgpt export
```

The result is written to `chatgpt-context-YYYY-MM-DD/`:

```text
chatgpt-context-2026-07-10/
├── README.md
├── manifest.json
├── duvo/
│   ├── AGENTS.md
│   ├── COWORK_HISTORY_INDEX.md
│   └── COWORK_HISTORY.md
├── another-workspace/
│   └── ...
└── _shared-memory/
    └── COWORK_SHARED_MEMORY.md
```

In Codex or ChatGPT desktop, open one workspace folder. For a browser ChatGPT Project, upload the
Markdown files from one portable workspace as project sources and set Project instructions in the
UI. Do not combine workspace folders unless you intentionally want their contexts mixed.

## Memory sources

For every selected workspace folder, the converter reads:

- `CLAUDE.md`
- Existing `MEMORY.md` or `memory.md`
- Existing `CONTEXT.md` or `context.md`
- Markdown files under `memory/`
- Structured `<user_preferences>` stored with relevant Cowork sessions

This project-specific guidance is placed in the managed section of `AGENTS.md`. If the combined
file would exceed Codex's documented default instruction limit, the imported portion moves to
`COWORK_INSTRUCTIONS.md` and `AGENTS.md` tells Codex to read it.

Hidden global Cowork memory may contain facts from several projects. It is never injected into an
original workspace automatically.

## Useful options

```bash
# Install one workspace shown by `scan`
cowork2chatgpt install --workspace duvo

# Only sessions active since a date
cowork2chatgpt install --since 2026-06-01

# Skip sessions archived in Cowork
cowork2chatgpt install --exclude-archived

# Add redacted, size-capped tool evidence and subagent history
cowork2chatgpt install --with-evidence

# Create portable packages instead of changing original folders
cowork2chatgpt export

# Omit hidden global memory from a portable export
cowork2chatgpt export ./context --no-shared-memory

# Use an explicit Cowork data location
cowork2chatgpt install --source "/path/to/local-agent-mode-sessions"
```

## Privacy model

| Data | Default | `--with-evidence` |
|---|---:|---:|
| User and assistant text | Included | Included |
| Workspace-owned memory | Included | Included |
| Tool calls and results | Excluded | Redacted and size-capped |
| Subagent history | Excluded | Included |
| Attachment descriptors | Excluded | Included |
| Hidden reasoning | Excluded | Excluded |
| System prompts | Excluded | Excluded |
| Raw metadata and JSONL | Excluded | Excluded |

Credential redaction is best effort, not a security guarantee. Installed memory and history may
be sensitive. Review them before uploading or sharing, and do not commit them to a public
repository. `COWORK_HISTORY_INDEX.md` reports what was included, omitted, truncated, or unreadable.

## Data location and support

On macOS, Cowork data is auto-detected at:

```text
~/Library/Application Support/Claude/local-agent-mode-sessions
```

Set `COWORK_DATA_DIR` or pass `--source` for custom locations. Only locally persisted sessions can
be converted. Cowork's local format is undocumented and may change, so schema reports should use
minimal synthetic fixtures, never real transcripts or memory.
